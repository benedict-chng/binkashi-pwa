**Part A: Request persistent storage (src/db/index.ts)**

After database initialization, request persistent storage:

```typescript
// Request persistent storage for Safari (prevents 7-day cache eviction)
if ('storage' in navigator && 'persist' in navigator.storage) {
  navigator.storage.persist().then((persistent) => {
    if (persistent) {
      console.log('Storage persistence granted');
    } else {
      console.log('Storage persistence denied');
    }
  }).catch((error) => {
    console.error('Failed to request persistent storage:', error);
  });
}
```

This prevents Safari from clearing IndexedDB after 7 days of inactivity.

**Part B: Add service worker update listener (src/App.tsx)**

Add useEffect in App component to listen for service worker updates:

```typescript
useEffect(() => {
  // Listen for service worker updates (custom event from vite-plugin-pwa)
  const handleUpdate = () => {
    // Show toast notification about new version
    // User can refresh to get update
    console.log('New version available');
    // Optionally use toast to notify user
  };

  // Listen for custom event dispatched by vite-plugin-pwa
  window.addEventListener('swUpdate', handleUpdate);

  return () => {
    window.removeEventListener('swUpdate', handleUpdate);
  };
}, []);
```

Note: vite-plugin-pwa may emit different event names. Check documentation for actual event name. Common events: 'swUpdate', 'swUpdated', 'sw-registration-success'.
