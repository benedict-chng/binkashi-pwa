# v1.1 Requirements to Verify

From ROADMAP.md, Phase 4 requirements:
- DES-01: App applies new colour palette to all UI elements (backgrounds, buttons, text, cards, containers)
- DES-02: Colour palette uses Black (#07020d), Sky Surge (#5db7de), Soft Linen (#f1e9db), Khaki Beige (#a39b8b), Dim Grey (#716a5c)
- DISP-01: Bin cards display "Days in Use" field
- DISP-02: Days in Use calculates days from inUseStartDate to current date
- DISP-03: Days in Use displays 0 when bin state is Empty
- UI-01: Toast notifications appear centered at top of screen (not top-right)
</context>

Create a comprehensive UAT checklist document at `.planning/phases/05-milestone-completion/05-UAT-CHECKLIST.md` that covers:

1. **Document Structure:**
   - Title: "v1.1 Milestone User Acceptance Testing Checklist"
   - Introduction: Purpose of testing, testing environment setup
   - Test Cases Section: Individual test cases for each requirement
   - Results Section: Table for recording test results
   - Sign-off Section: User approval

2. **Test Cases** (one per requirement):

   **DES-01: Colour Palette Application**
   - Test Step 1: Load app in browser (http://localhost:4173 or deployed URL)
   - Expected Result: App displays with Soft Linen background (#f1e9db), all UI elements use new colors
   - Verification Method: Visual inspection - backgrounds, buttons, text, cards, containers all use new palette

   **DES-02: Specific Color Values**
   - Test Step 1: Inspect specific UI elements using DevTools or visual inspection
   - Expected Result:
     - Main text: Black (#07020d)
     - Secondary text: Dim Grey (#716a5c)
     - Buttons: Sky Surge (#5db7de)
     - Backgrounds: Soft Linen (#f1e9db)
     - Borders: Khaki Beige (#a39b8b)
   - Verification Method: Compare against color values in src/index.css

   **DISP-01: Days in Use Field Display**
   - Test Step 1: Create a new bin with name "Test Bin" and set state to "In Use"
   - Expected Result: Bin card displays "Days in Use: X" field where X is a number
   - Verification Method: Visual inspection of bin card after creation

   **DISP-02: Days in Use Calculation**
   - Test Step 1: Create bin with inUseStartDate set to 10 days ago (adjust accordingly)
   - Expected Result: "Days in Use" shows the correct number of days (e.g., 10)
   - Verification Method: Compare displayed days with actual date difference

   **DISP-03: Days in Use for Empty State**
   - Test Step 1: Create bin and set state to "Empty"
   - Expected Result: "Days in Use: 0" displays regardless of date values
   - Verification Method: Visual inspection shows "Days in Use: 0"

   **UI-01: Toast Notification Positioning**
   - Test Step 1: Create a bin to trigger success toast, or perform any action that shows a toast
   - Expected Result: Toast appears centered horizontally at the top of the screen
   - Verification Method: Visual inspection - toast should be centered (not top-right)

3. **Results Table:**
   Create a table with columns: Test Case, Test Step, Expected Result, Actual Result, Status (Pass/Fail), Notes

4. **Pre-Testing Checklist:**
   - App built successfully (npm run build)
   - App previewed successfully (npm run preview or deployed)
   - Browser: Chrome/Safari/Edge/Firefox (specify which)
   - Device: Mobile/Tablet/Desktop (specify which)

The document should be clear, actionable, and executable by a user without technical knowledge of the codebase.
